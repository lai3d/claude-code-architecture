# Claude Code CLI v2.1.88 — 整体架构概览

## 1. 项目概况

Claude Code CLI 是一个基于 **TypeScript + React (Ink)** 的终端应用程序，使用 **Bun** 作为运行时和打包工具。源代码约包含 **~1900 个文件**，实现了一个功能完备的 AI 对话式编程助手。

核心技术栈：

| 技术 | 用途 |
|------|------|
| TypeScript | 主语言 |
| React (Ink) | 终端 UI 框架 |
| Bun | 运行时 / 打包器 |
| Commander.js | CLI 参数解析 |
| MCP 协议 | 可扩展工具系统 |
| Yoga (Flexbox) | 终端布局引擎 |

## 4. 核心模块详解

### 4.1 入口层 (`entrypoints/`)

应用有多个入口点，适配不同的使用场景：

- **`cli.tsx`** — 标准 CLI 入口，执行引导流程（bootstrap），初始化环境
- **`init.ts`** — 项目初始化入口（`claude init`）
- **`mcp.ts`** — 作为 MCP 服务器运行时的入口
- **`sdk/`** — 程序化 SDK 入口，供外部程序集成

### 4.2 CLI 解析与会话建立 (`main.tsx`)

`main.tsx` 是所有入口的汇聚点，负责：

1. 使用 Commander.js 解析命令行参数
2. 加载配置和用户设置
3. 建立会话上下文
4. 根据运行模式分流：
   - **REPL 模式** — 交互式终端，渲染 `screens/REPL.tsx`
   - **Headless 模式** — 单次查询，直接调用 QueryEngine
   - **SDK 模式** — 程序化接口

### 4.3 核心对话引擎 (`QueryEngine.ts`)

`QueryEngine` 是整个应用的心脏，实现了对话循环的核心逻辑：

```
用户输入 → 构建消息 → 调用 API → 解析响应 → 执行工具 → 返回结果
    ↑                                                    │
    └────────────────── 循环 ──────────────────────────────┘
```

关键设计：

- **AsyncGenerator 模式** — 使用异步生成器流式产出结果，UI 层可以实时渲染
- **工具执行循环** — 当模型返回 tool_use 时，自动执行工具并将结果送回模型
- **对话压缩** — 通过 `services/compact/` 在对话过长时自动压缩上下文

### 4.4 API 调用层 (`query.ts` + `services/api/`)

负责与 Claude API 的通信：

- 构建请求参数（模型选择、系统提示词、工具定义）
- 处理流式响应 (SSE)
- 错误处理与重试
- 支持多种 API 提供者

### 4.5 工具系统 (`Tool.ts` / `tools.ts` / `tools/`)

工具系统是 Claude Code 最核心的能力层：

```
┌──────────────────────────────────────────────────────┐
│                   tools.ts (注册表)                    │
│          feature flag 条件加载 + MCP 动态注册          │
├──────────────────────────────────────────────────────┤
│                                                      │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌────────┐  │
│  │ BashTool │ │ EditTool │ │ ReadTool │ │GrepTool│  │
│  └──────────┘ └──────────┘ └──────────┘ └────────┘  │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌────────┐  │
│  │ GlobTool │ │WriteTool │ │AgentTool │ │WebFetch│  │
│  └──────────┘ └──────────┘ └──────────┘ └────────┘  │
│  ┌──────────────────┐ ┌──────────────────────────┐   │
│  │ NotebookEditTool │ │ MCP 动态工具 (外部扩展)   │   │
│  └──────────────────┘ └──────────────────────────┘   │
│                    ... 40+ 工具                       │
└──────────────────────────────────────────────────────┘
```

**关键类型定义 (`Tool.ts`)**：

- **`Tool`** — 工具接口，定义名称、描述、参数 Schema、执行函数
- **`ToolUseContext`** — 工具执行上下文，包含权限信息、会话状态
- **权限类型** — 三种权限模式：
  - `default` — 危险操作需用户确认
  - `auto` — 自动批准所有操作
  - `bypass` — 绕过权限检查

**条件加载**：`tools.ts` 通过 `feature()` 宏实现编译期死代码消除，未启用的工具不会进入最终产物。

### 4.6 UI 层 (`components/` + `screens/` + `ink/`)

UI 采用 React (Ink) 架构，支持组件化和 Flexbox 布局：

- **`ink/`** — 深度定制的 Ink 框架分支，包含自定义布局引擎、Yoga 集成、焦点管理、鼠标支持
- **`screens/`** — 顶层屏幕组件
  - `REPL.tsx` — 主交互界面
  - `Doctor.tsx` — 诊断界面
  - `ResumeConversation.tsx` — 恢复会话界面
- **`components/`** — 200+ 可复用 UI 组件（消息展示、输入框、工具输出、进度指示等）

### 4.7 状态管理 (`state/`)

使用 `AppStateStore` 进行集中式状态管理，包括：

- 当前对话状态
- 用户配置
- 工具状态
- UI 状态

## 5. 关键架构决策

### 5.1 React (Ink) 用于终端 UI

选择 React 组件模型来构建终端界面，带来了：

- **组件化** — 复杂 UI 拆分为可复用组件
- **Flexbox 布局** — 通过 Yoga 引擎实现灵活的终端布局
- **声明式渲染** — 状态变化自动驱动 UI 更新
- **深度定制** — 分叉 Ink 框架以支持焦点管理、鼠标事件等高级功能

### 5.2 Bun 打包与 `feature()` 编译期优化

利用 Bun 的打包能力和 `feature()` 宏实现：

- **编译期死代码消除** — 未启用的功能模块不进入最终产物
- **条件工具加载** — 根据 feature flag 决定哪些工具可用
- **减小产物体积** — 只打包实际需要的代码

### 5.3 MCP 协议扩展工具

通过 Model Context Protocol (MCP) 实现工具系统的可扩展性：

- 内置工具通过 `tools/` 目录注册
- 外部工具通过 MCP 服务器动态注册
- 统一的工具接口和权限模型

### 5.4 AsyncGenerator 流式处理

QueryEngine 使用 AsyncGenerator 模式：

- 支持流式产出对话结果
- UI 可以实时渲染每个 token
- 自然支持中断和取消操作
- 优雅处理工具执行循环

### 5.5 多层权限系统

权限控制贯穿整个工具执行链路：

```
用户操作 → 权限检查 → 工具执行
              │
    ┌─────────┼──────────┐
    ▼         ▼          ▼
 default    auto      bypass
 (需确认)  (自动批准)  (跳过检查)
```

## 6. 数据流概览

一次典型的用户交互流程：

```
1. 用户输入
   │
2. REPL.tsx 捕获输入
   │
3. 检查是否为斜杠命令 (/commands)
   │     是 → commands/ 处理
   │
4. QueryEngine.query() 启动对话循环
   │
5. 构建消息上下文 (系统提示词 + 历史 + 用户消息)
   │
6. query.ts → API 调用 (流式)
   │
7. 解析响应
   │
   ├── 文本内容 → 流式渲染到 UI
   │
   └── tool_use → 权限检查 → 工具执行 → 结果送回 → 回到步骤 6
   │
8. 对话结束，更新状态，等待下一次输入
```

## 8. 总结

Claude Code CLI 的架构围绕 **对话循环** 这一核心展开，通过分层设计将入口、解析、引擎、工具、UI 清晰分离。React (Ink) 的引入使得终端 UI 具备了现代前端的组件化能力，而 MCP 协议和插件系统则保证了工具层的可扩展性。AsyncGenerator 流式处理和多层权限系统是两个贯穿全局的关键设计模式，前者保证了用户体验的实时性，后者保证了操作的安全性。
